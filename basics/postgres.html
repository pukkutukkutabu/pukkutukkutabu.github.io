# Postgres

Разлисные сниппеты для postgres (9.6, 11)

# Репликация

Репликация состоит из 2х шагов, создание слота репликации на сервере с которого она будет осуществляться и репликации (копирования) данных на новую реплику с помощью программы pg_basebackup

На сервере с которого будет осуществяться репликация (мастер или реплика):

SELECT pg_create_physical_replication_slot('slot_name');

На новом сервере:

#basebackup from postgres user (centos)
time pg_basebackup -P -R -X stream -S 'slot_name' -c fast -h {{ip_address}} -U postgres -D /var/lib/pgsql/9.6/data/ -p 5432

где - ip адрес сервера

опции:

  • -P -

  • -X - тип репликации (в данном случае stream)

  • -S - имя слота

  • -c fast - не дожидаться следующего checkpoint'a на сервере, а сделать сразу

  • -R - создавать recovery.conf (перетирает если на сервере уже имеется)
